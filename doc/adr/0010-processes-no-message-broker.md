# ADR-0010: Запуск — два процесса, без брокера очередей

## Статус
Принято

## Контекст
Нужно разделить API и бота; решить, вводить ли очереди (Redis/RabbitMQ) на MVP.

## Решение
- **Два процесса**: API (FastAPI) и bot (aiogram).
- **Без брокера очередей** (Redis/RabbitMQ не используем на MVP).
- При необходимости периодических задач — простой scheduler или async-task внутри API.

## Последствия
- Проще деплой и отладка; логи и рестарты изолированы по процессам.
- Фоновые задачи — минимально, внутри одного процесса; при росте нагрузки можно ввести очередь позже.
