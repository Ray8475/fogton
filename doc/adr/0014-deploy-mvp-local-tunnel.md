# ADR-0014: Деплой MVP — через GitHub, локально — для отладки

## Статус
Принято (обновлено: сервер больше не разворачиваем через Cloudflare)

## Контекст
Нужен способ дать доступ к боту и API по HTTPS для webhook Telegram и TON; для продакшена — воспроизводимый деплой из репозитория.

## Решение
- **Основной деплой:** через **GitHub** (GitHub Actions → хостинг/VPS; webapp → GitHub Pages). Сервер не разворачиваем через Cloudflare Tunnel.
- **Локальная отладка:** при необходимости — локальный запуск API и бота + туннель (ngrok или другой) для публичного HTTPS; Mini App локально (`http.server`) или через Pages.
- Mini App в продакшене — статика на GitHub Pages (workflow уже есть); главное — доступ по HTTPS.

## Последствия
- Один репозиторий, деплой из GitHub; статика по HTTPS через Pages.
- Для разработки можно использовать локальный запуск и туннель по желанию; для продакшена — хостинг, подключаемый к GitHub (VPS, и т.п.).
