# ADR-0007: Платежи/депозиты — кастодиальная модель, один кошелёк + comment

## Статус
Принято

## Контекст
Нужна простая модель зачисления депозитов пользователям без отдельных кошельков на пользователя на MVP.

## Решение
**Кастодиальная модель**:
- Депозиты приходят на **один кошелёк проекта**.
- Пользователю начисляется **внутренний баланс** в приложении.
- **Атрибуция депозита пользователю (MVP)**: один кошелёк проекта + **уникальный комментарий/пэйлоад** (memo) в переводе.

Опционально позже: уникальный депозитный адрес на пользователя.

## Последствия
- Один кошелёк — проще управление и мониторинг.
- Пользователь обязан указывать правильный comment при переводе; при ошибке — ручная атрибуция или поддержка.
