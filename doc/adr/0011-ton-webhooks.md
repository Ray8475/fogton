# ADR-0011: TON — приём событий через webhook

## Статус
Принято

## Контекст
Нужно получать уведомления о входящих переводах TON/USDT на кошелёк проекта для зачисления баланса.

## Решение
- Провайдер TON API отправляет **webhook** на наш API (отдельный endpoint).
- API валидирует событие, сопоставляет пользователя по comment/payload, создаёт запись депозита и увеличивает внутренний баланс.

## Последствия
- Не нужен polling блокчейна; провайдер сам шлёт события.
- Нужен публичный HTTPS для приёма webhook; валидация подписи/секрета — по возможностям провайдера.
