# ADR-0006: Аутентификация — Telegram WebApp initData → JWT

## Статус
Принято

## Контекст
Mini App должна надёжно идентифицировать пользователя при обращении к API без передачи секретов.

## Решение
- Backend проверяет подпись **Telegram WebApp initData** и извлекает идентификатор пользователя.
- После успешной проверки выдаётся **короткоживущий JWT** (например 1–6 часов).
- Дальше Mini App ходит в API с заголовком `Authorization: Bearer <jwt>`.

## Последствия
- Не нужно хранить пароли; доверяем только подписанным данным Telegram.
- JWT упрощает проверку на каждом запросе; срок жизни ограничивает риски при утечке.
