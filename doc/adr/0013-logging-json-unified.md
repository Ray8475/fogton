# ADR-0013: Логгирование — JSON, единая структура

## Статус
Принято

## Контекст
Нужны читаемые и машинообрабатываемые логи во всех процессах без сложного мониторинга на MVP.

## Решение
- **JSON-логи** во всех процессах (api, bot) с одинаковым набором полей.
- Минимальный набор: `timestamp`, `level`, `service` (api|bot), `event`, `message`; при необходимости `request_id`, `telegram_user_id`, `tx_hash`.
- Ошибки — с stacktrace.

## Последствия
- Удобно искать и фильтровать (например, по `service`, `request_id`).
- Единый формат упрощает добавление лог-агрегации позже.
